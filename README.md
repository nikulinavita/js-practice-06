## Ответы на контрольные вопросы
1. **Чем `querySelector` отличается от `querySelectorAll`? Что возвращает второй метод?**
   `querySelector` находит и возвращает только первый совпавший элемент на странице. `querySelectorAll` находит все подходящие элементы и возвращает `NodeList` (статическую коллекцию узлов).

2. **Почему в этой работе мы отдаем предпочтение `textContent`, а не `innerHTML`?**
   В целях безопасности (защита от XSS-атак). `textContent` вставляет пользовательские данные как обычный текст, а `innerHTML` может выполнить вредоносный код, если в строке окажутся HTML-теги или скрипты.

3. **Как работает метод `classList.toggle()`? Опишите логику.**
   Метод переключает состояние: проверяет наличие класса у элемента. Если класс есть, он его удаляет. Если класса нет, он его добавляет.

4. **Что такое «узел» (node) и «элемент» (element) в контексте DOM?**
   Узел — это абсолютно любой объект в DOM-дереве (текст, комментарий, тег). Элемент - это конкретный тип узла, представляющий собой HTML-тег (например, `<div>` или `<p>`). Каждый элемент является узлом, но не каждый узел - элементом.

5. **Зачем нужен `DocumentFragment` и в каких случаях его использование оправдано?**
   Это легковесный контейнер для сборки элементов в оперативной памяти (вне активного документа). Его использование оправдано при массовой вставке множества узлов (например, 50 карточек разом), чтобы браузер перерисовал страницу только один раз, сэкономив ресурсы.
